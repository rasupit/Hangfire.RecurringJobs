# Hangfire.Extension.AspNetCore v0.1.0

First public release of `Hangfire.Extension.AspNetCore`.

## Highlights

- Embedded recurring jobs admin UI for ASP.NET Core Razor Pages hosts
- Custom route mapping with `MapHangfireExtension(...)`
- Single runtime library assembly for simpler integration
- Built-in recurring job definition registry for enable/update operations
- Paged recurring jobs API with search metadata
- `SkipConcurrentExecutionAttribute` for overlap protection
- Host-level integration tests and SQLite-backed Hangfire integration tests

## Installation

Add the package to an existing ASP.NET Core app that already uses Hangfire:

```powershell
dotnet add package Hangfire.Extension.AspNetCore --version 0.1.0
```

## Usage

Register the package after your Hangfire setup:

```csharp
using Hangfire;
using Hangfire.Extension.AspNetCore.DependencyInjection;
using Hangfire.Extension.AspNetCore.Routing;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddAuthorizationBuilder()
    .AddPolicy("RecurringJobAdmin", policy => policy.RequireRole("Admin"));

builder.Services.AddHangfire(configuration =>
{
    configuration.UseSimpleAssemblyNameTypeSerializer();
    configuration.UseRecommendedSerializerSettings();

    // Reuse your existing Hangfire storage here.
});

builder.Services.AddHangfireExtension("/recurring-jobs");

var app = builder.Build();

app.UseAuthorization();

app.MapHangfireExtension("/recurring-jobs");
app.MapRazorPages();

app.Run();
```

The embedded UI will be available at:

- `/recurring-jobs`

The embedded API will be available under:

- `/recurring-jobs/api/jobs/recurring`

## Optional: Register Code-Based Definitions

Enable and cron-update actions need a code-based definition source.

```csharp
builder.Services.AddRecurringJobDefinition<ReportJobs>(
    "nightly-report",
    jobs => jobs.RunNightlyReport(),
    "0 2 * * *",
    TimeZoneInfo.Utc,
    "default");
```

## Optional: Prevent Overlapping Executions

```csharp
using Hangfire.Extension.Core.Filters;

public sealed class ReportJobs
{
    [SkipConcurrentExecution(timeoutInSeconds: 30)]
    public Task RunNightlyReport()
        => Task.CompletedTask;
}
```

## Notes

- This package does not provide a default Hangfire storage.
- It is intended to be added to an app that already has Hangfire installed and configured.
- The sample host in `src/Hangfire.Extension.Web` is for reference and testing.
