# Hangfire.Extension v0.1.6

Patch release for self-contained recurring jobs styling.

## Highlights

- Restores self-contained recurring jobs page styling by inlining the stylesheet from an embedded library resource
- Removes the runtime dependency on package-scoped CSS requests for the recurring jobs admin surface
- Preserves the refreshed UI introduced in the previous release without requiring host static asset mapping for page styling

## Installation

```powershell
dotnet add package Hangfire.Extension --version 0.1.6
```

## Consumer Impact

- No host integration changes are required
- Hosts should continue to call `MapHangfireRecurringJobsApi()` and `MapRazorPages()`
- Recurring jobs styling no longer depends on successful requests to `hangfire-extension.css` or Bootstrap CSS assets

## Notes

- This package still does not configure Hangfire storage
- The recurring jobs UI remains intended for internal/admin usage
