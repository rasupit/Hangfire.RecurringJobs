# Hangfire.Extension v0.1.11

Patch release for recurring jobs stylesheet delivery reliability.

## What's Changed

- Removes the separate runtime Bootstrap stylesheet dependency from the recurring jobs layout
- Restores the embedded fallback theme so the default library UI does not depend on external stylesheet asset paths
- Keeps `options.Styles` as the host override hook when a consumer wants to supply its own site theme
- Expands the fallback stylesheet so it fully styles the current recurring jobs pages on its own

## Host Integration

Hosts should continue to call:

- `MapStaticAssets()`
- `MapHangfireRecurringJobsApi()`
- `MapRazorPages()`

If `Styles` is not configured, the library uses its embedded fallback theme automatically.

If `Styles` is configured, the library emits only that stylesheet link:

```csharp
builder.Services.AddHangfireRecurringJobs(options =>
{
    options.RoutePrefix = "/jobs";
    options.RequireAuthorization = true;
    options.Styles = "/css/site.css";
});
```
