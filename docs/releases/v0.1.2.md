# Hangfire.Extension v0.1.2

Patch release for embedded layout isolation.

## Highlights

- Fixes the recurring jobs UI so host applications do not override the library layout
- Ensures the embedded stylesheet link is rendered from the library-owned layout
- Adds host integration coverage to prevent future layout collisions

## Installation

```powershell
dotnet add package Hangfire.Extension --version 0.1.2
```

## Consumer Impact

- No host changes are required
- Hosts should continue to call `MapStaticAssets()` and `MapRazorPages()`
- The embedded UI now consistently uses the library layout even when the host has its own `/Pages/Shared/_Layout.cshtml`

## Notes

- This package still does not configure Hangfire storage
- The recurring jobs UI remains intended for internal/admin usage
