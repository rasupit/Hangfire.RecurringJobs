# Hangfire.RecurringJobs v0.1.33

## Bug fixes

- Fixed mobile navigation: navbar toggle was always hidden because the `navbar-expand-sm` rules were missing their `@media (min-width: 576px)` wrapper.
- Fixed CSS `@media` block order: the `min-width: 768px` container rule appeared after the 992px and 1200px blocks, causing the container max-width to be capped too narrow on wide screens.
- Fixed `SkipConcurrentExecutionAttribute` lock key collision: the distributed lock is now scoped to the recurring job ID (`RecurringJobId` Hangfire parameter) instead of the CLR type and method name, preventing unrelated jobs from blocking each other.

## Improvements

- `Next:` and `Last:` execution timestamps are now shown in each job's own registered timezone (sourced from `RecurringJobDto.TimeZoneId` — the IANA ID stored by Hangfire at job registration). Seconds are omitted from all datetime displays for readability.
- Cron preview upcoming occurrences (inline dialog and full Edit page) now compute in the job's registered timezone, keeping the schedule description and occurrence times consistent.
- Job column now word-wraps long CLR type names, eliminating horizontal scroll on the index table.
- Schedule column width increased to better balance the table layout.
- Removed the `DisplayTimeZone` option from `RecurringJobsOptions` — the job's own registered timezone is the single source of truth and requires no additional configuration.
- Removed dead `RecurringJobStorage.GetJobsAsync(RecurringJobQuery)` overload and its orphaned private search helpers.
- Edit page now preserves the search filter and page number when navigating back to the index.
