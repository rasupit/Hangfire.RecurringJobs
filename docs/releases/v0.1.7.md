# Hangfire.Extension v0.1.7

Patch release for recurring jobs layout cleanup.

## Highlights

- Removes the extra secondary callout from the recurring jobs header
- Aligns the search actions beside the search box for a tighter toolbar layout
- Shrinks row action buttons so trigger, enable or disable, and edit actions fit cleanly at the end of each row

## Installation

```powershell
dotnet add package Hangfire.Extension --version 0.1.7
```

## Consumer Impact

- No host integration changes are required
- Hosts should continue to call `MapHangfireRecurringJobsApi()` and `MapRazorPages()`
- The recurring jobs page keeps the self-contained inline styling introduced in the previous release

## Notes

- This package still does not configure Hangfire storage
- The recurring jobs UI remains intended for internal/admin usage
