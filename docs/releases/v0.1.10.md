# Hangfire.Extension v0.1.10

Patch release for the Bootstrap-based recurring jobs UI refresh and configurable theme stylesheet.

## What's Changed

- Switches the recurring jobs pages to standard Bootstrap markup for forms, cards, tables, alerts, buttons, and pagination
- Adds `RecurringJobsOptions.Styles` so hosts can point the UI at their own site theme stylesheet
- Loads packaged Bootstrap and falls back to `/hangfire-extension/css/hangfire-extension.css` when no custom stylesheet is configured
- Keeps the route prefix as the single source of truth for the UI and API integration pattern

## Host Integration

Hosts should continue to call:

- `MapStaticAssets()`
- `MapHangfireRecurringJobsApi()`
- `MapRazorPages()`

To use a host theme instead of the packaged fallback stylesheet:

```csharp
builder.Services.AddHangfireRecurringJobs(options =>
{
    options.RoutePrefix = "/jobs";
    options.RequireAuthorization = true;
    options.Styles = "/css/site.css";
});
```
