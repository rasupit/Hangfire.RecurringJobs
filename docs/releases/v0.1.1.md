# Hangfire.Extension v0.1.1

Patch release for static web asset packaging.

## Highlights

- Moves package-owned static assets under `/hangfire-extension/...`
- Prevents collisions with host application assets such as `lib/bootstrap`, `lib/jquery`, `css/...`, and `favicon.ico`
- Keeps consumer integration unchanged aside from the existing `MapStaticAssets()` requirement
- Adds host integration coverage for the packaged asset path

## Installation

```powershell
dotnet add package Hangfire.Extension --version 0.1.1
```

## Consumer Impact

- No host `.csproj` exclusions are required for this fix
- Hosts should continue to call `MapStaticAssets()` and `MapRazorPages()`
- The embedded UI now serves its assets from `/hangfire-extension/...`

## Notes

- This package still does not configure Hangfire storage
- The recurring jobs UI remains intended for internal/admin usage
