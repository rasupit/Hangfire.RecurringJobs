# Hangfire.Extension v0.1.5

Patch release for the refreshed recurring jobs UI and repo-local package restore.

## Highlights

- Refreshes the embedded recurring jobs UI with a cleaner layout, stronger hierarchy, and improved mobile behavior
- Moves recurring jobs page styling to the package-scoped stylesheet instead of inline page CSS
- Configures restore to use the repository-local NuGet package cache so local builds do not depend on the roaming user profile path

## Installation

```powershell
dotnet add package Hangfire.Extension --version 0.1.5
```

## Consumer Impact

- No integration API changes are required
- Hosts should continue to call `MapStaticAssets()`, `MapHangfireRecurringJobsApi()`, and `MapRazorPages()`
- Consumers using a local package feed can update the package reference and restore normally

## Notes

- This package still does not configure Hangfire storage
- The recurring jobs UI remains intended for internal/admin usage
